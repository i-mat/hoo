Create github blog by jekyll

Andy Sun 23 Mar 2020

1,Jekyll introduction

the web site is https://jekyllrb.com

It transform your plain text , such as markdown file , into static web pages.

Since github offer us the possibility that we build our own static web site , jekyll become to be a very useful tool.

There are a lot of themes for us to us .

https://jamstackthemes.dev/ssg/jekyll/

http://jekyllthemes.org/

http://jekyllthemes.org/

For sure , some themes is not for free , and perhaps you think all of them cannot please you.

To create one theme yourself seems to make great sense !

2,Convert static html to jekyll theme by yourself

At first, you should learn how jekyll works . https://jekyllrb.com/docs/

Second , please pay attention to this blog , it show us how to convert static html to jekyll . https://jekyllrb.com/tutorials/convert-site-to-jekyll/

Finally , try to build the demo theme based on one static web page you like .

Trust me , once you get there , that is pretty cool !

3,some keypoints

A, pagination

if you perfer checking all the blogs at one web page , plugin jekyll-paginate is enough for you .

But the fact is , we need categories pagination usually , please check this git repo https://github.com/luis-almeida/jPages ,integrate jPage.js into your theme , you could check my code to find how I customize it and use it .

B, comment plugin

Do you like someone who viewed your blog to comment under your blog? Good advice make great sense .

It’s cost-effective to develop this function yourself , on the contrary , to integrate one comment plugin make us happy .

https://disqus.com/

4,clone my customize theme

https://github.com/CrazyAndy/NewWorldBlogTemplate

My advice is to convert one of your favorite static web site to your own theme , and create markdown file under the folder _posts of your theme . It will help you see a strange wonderful world .

5,push your html which is under folder _site to GitHub

Google github page to get to know how to build it . It’s so easy !
